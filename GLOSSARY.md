# 📖 용어 사전 (GLOSSARY.md)

> 이 파일은 프로젝트에서 사용되는 개발 용어를 쉽게 정리한 사전입니다.
> Claude Code가 새로운 용어를 사용할 때 자동으로 여기에 추가합니다.
> 직접 읽어보면서 복습하면 실력이 늘어요!

## 기본 용어

| 용어 | 쉬운 설명 | 👤 유저 입장에서 보면 |
|------|-----------|----------------------|
| API | 프로그램끼리 대화하는 창구 | 앱에서 새로고침하면 로딩 도는 동안 API가 서버에 "데이터 줘" 하고 있는 것 |
| CLI | 키보드로 명령어를 입력하는 검은 화면 | Claude Code가 돌아가는 그 터미널 화면 |
| UI | 사용자가 보고 클릭하는 화면 | 앱 켜면 보이는 버튼, 메뉴, 카드 같은 것들 전부 |
| 컴포넌트 | 화면을 구성하는 레고 블록 같은 부품 | 유튜브의 "영상 카드" 하나하나가 컴포넌트. 똑같은 틀에 다른 내용만 바뀜 |
| props | 부모 컴포넌트가 자식에게 전달하는 데이터 | 유튜브 영상 카드에 "제목, 썸네일, 조회수"를 넘겨주는 것 |
| state | 컴포넌트가 기억하는 현재 상태 값 | 인스타 좋아요 버튼의 "눌림/안눌림" 상태. 누르면 바로 하트 색이 바뀌는 이유 |
| import | 다른 파일에서 코드를 가져오는 것 | 앱의 설정 화면에서 "프로필 사진 컴포넌트"를 다른 데서 가져다 쓰는 것 |
| export | 이 파일의 코드를 다른 곳에서 쓸 수 있게 내보내는 것 | 내가 만든 버튼을 여러 페이지에서 재사용할 수 있게 공개하는 것 |
| 라우팅 | URL 주소에 따라 다른 페이지를 보여주는 것 | 유튜브에서 홈/구독/보관함 탭 누르면 주소가 바뀌면서 다른 화면이 나오는 것 |
| 배포 (deploy) | 만든 걸 인터넷에 올려서 누구나 접속하게 하는 것 | "내 폰에서만 되던 게 링크 공유하면 남들도 볼 수 있게 되는 순간" |
| 빌드 (build) | 코드를 브라우저가 이해할 수 있게 변환하는 것 | 배포 전에 하는 준비 작업. 안 하면 브라우저가 코드를 못 읽음 |
| 디버깅 | 코드에서 문제를 찾아서 고치는 것 | 앱이 갑자기 멈출 때 "왜 멈췄지?" 원인을 추적하는 과정 |
| 에러 (error) | 코드 실행 중 뭔가 잘못됐다는 신호 | 앱에서 "문제가 발생했습니다" 뜨는 그 순간, 뒤에서 에러가 터진 것 |
| 콘솔 (console) | 개발자가 디버깅 메시지를 확인하는 곳 | 브라우저에서 F12 누르면 나오는 창. 에러 메시지가 여기에 빨간 글씨로 뜸 |
| 의존성 (dependency) | 내 프로젝트가 사용하는 외부 라이브러리 | 카카오톡이 "카메라 앱"에 의존하는 것처럼, 내 앱도 남이 만든 도구를 가져다 씀 |
| 환경변수 (.env) | 비밀번호 등 공개하면 안 되는 설정값 | 앱 로그인할 때 내 비번이 코드에 적혀있으면 안 되니까 따로 보관하는 것 |
| 리렌더링 | 화면이 다시 그려지는 것 | 인스타 피드 당겨서 새로고침하면 화면이 다시 그려지는 것. state가 바뀔 때도 자동으로 일어남 |
| 콜백 (callback) | "이거 끝나면 이걸 해줘"라고 예약해두는 함수 | 배달앱에서 "배달 완료되면 알림 보내줘"와 같은 원리 |
| 비동기 (async) | 결과를 기다리지 않고 다른 일을 먼저 하는 방식 | 카페에서 주문 넣고 진동벨 받고 자리에서 기다리는 것. 주문 넣고 카운터 앞에서 멈춰 서는 게 동기 |
